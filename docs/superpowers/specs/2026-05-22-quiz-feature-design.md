# Quiz Feature — "Test Yourself"
**Date:** 2026-05-22
**Status:** Approved for implementation

---

## Overview

A quiz feature accessible from the Vault section, allowing users to test their knowledge of either Approved Documents (building regulations) or the CITB CSCS Health & Safety test. Questions are pre-generated by Gemini and stored in Supabase. Results are tracked per user and visible to admins only.

---

## Entry Point

A small grey outline button labelled **"✎ Test Yourself"** is added to the vault toolbar, sitting next to the Re-index button. It is always visible in the vault section regardless of which vault is open.

**Style:** `border: 1px solid #d0ccc8; color: #7a7a80;` — understated, does not compete with primary vault controls.

---

## Quiz Modal Flow

### Screen 1 — Subject Selection
Two tiles displayed side by side:
- **Approved Documents** (teal, VAULT_FULL) — "Building regulations questions drawn from your vault"
- **CITB CSCS** (slate, TIMESHEETS_FULL) — "Health, safety & environment test questions"

No stats are shown on the tiles. Users see subject tiles only.

### Screen 2a — Document Picker (Approved Documents path only)
A list of all documents in the dedicated AD vault, each showing:
- Document name
- Number of available questions (e.g. "42 questions →")

User selects a document to begin. If a document has no questions yet (not yet generated), it is shown greyed out with "No questions yet".

### Screen 2b — CSCS path
Skips the document picker. Goes directly to the quiz using the full CSCS question bank.

### Screen 3 — Quiz
- One question displayed at a time
- Four answer options labelled A / B / C / D
- On selection:
  - **Correct:** selected option turns green with ✓
  - **Wrong:** selected option turns red with ✗, correct option highlighted green, brief explanation shown below
- "Next Question →" button appears after answering
- Questions are served in random order, cycling indefinitely until the modal is closed
- No session score shown to the user — just per-question feedback

## Data Model

### `quiz_questions` table
```sql
CREATE TABLE IF NOT EXISTS quiz_questions (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  type text NOT NULL CHECK (type IN ('approved_docs', 'cscs')),
  vault_id uuid REFERENCES vaults(id) ON DELETE CASCADE,
  document_name text NOT NULL,
  question_text text NOT NULL,
  options jsonb NOT NULL,
  -- options format: [{ "label": "A", "text": "...", "is_correct": true }, ...]
  explanation text,
  source_clause text,
  created_at timestamptz NOT NULL DEFAULT now()
);
ALTER TABLE quiz_questions ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Auth read" ON quiz_questions FOR SELECT USING (true);
CREATE POLICY "Admin write" ON quiz_questions FOR ALL USING (true) WITH CHECK (true);
```

### `quiz_stats` table
```sql
CREATE TABLE IF NOT EXISTS quiz_stats (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  quiz_type text NOT NULL CHECK (quiz_type IN ('approved_docs', 'cscs')),
  correct_count int NOT NULL DEFAULT 0,
  incorrect_count int NOT NULL DEFAULT 0,
  updated_at timestamptz NOT NULL DEFAULT now(),
  UNIQUE (user_id, quiz_type)
);
ALTER TABLE quiz_stats ENABLE ROW LEVEL SECURITY;
-- Users can upsert their own stats (increment counters), admins can read all
CREATE POLICY "Own write" ON quiz_stats FOR ALL USING (auth.uid() = user_id);
```

One row per user per quiz type. On each answer, the server increments the appropriate counter via upsert.

## Server Endpoints

All new endpoints in `server/index.js`:

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/quiz/questions` | Fetch questions — params: `type`, `vault_id` (optional), `document_name` (optional) |
| `POST` | `/api/admin/quiz/generate` | Generate questions for one AD document via Gemini. Body: `{ vault_id, document_name }` |
| `DELETE` | `/api/admin/quiz/questions` | Clear questions for a document. Body: `{ vault_id, document_name }` |
| `POST` | `/api/admin/quiz/upload-cscs` | Upload + parse CSCS PDF, store questions verbatim |
| `POST` | `/api/quiz/answer` | Record an answer. Body: `{ quiz_type, is_correct }`. Upserts user's stat row |
| `GET` | `/api/admin/quiz/stats` | Admin only — returns all users' correct/incorrect counts for AD and CSCS |

`requireAdmin` middleware applied to all `/api/admin/quiz/*` routes.

## Question Generation — Approved Documents

When admin clicks "Generate" for a document:
1. Server fetches the document's extracted text (already indexed in vault)
2. Sends to Gemini with a prompt asking for 20–30 multiple choice questions, each with 4 options, one correct answer, a brief explanation, and the source clause reference
3. Gemini returns JSON array of questions
4. Questions stored in `quiz_questions` with `type = 'approved_docs'`
5. Existing questions for that document are cleared first (regenerate = fresh slate)

---

## CSCS Upload — Admin Section

Admin uploads the CSCS PDF in the admin Quiz tab. Server:
1. Extracts text using mupdf (same pattern as vault indexing)
2. Parses the verbatim Q&A format to extract question, options, and correct answer
3. Stores in `quiz_questions` with `type = 'cscs'`, `document_name = 'CSCS Health & Safety Test'`
4. Existing CSCS questions are cleared before inserting new ones

---

## AD Vault Selection

The quiz system needs to know which vault is the "Approved Documents" source. In the admin Quiz tab, a dropdown lets the admin designate one vault as the AD quiz vault. This selection is stored in a simple `app_settings` table (key: `quiz_ad_vault_id`, value: vault uuid). The quiz modal always fetches AD documents from that designated vault. If no vault is designated yet, the Approved Documents tile shows a "Not configured" state.

```sql
CREATE TABLE IF NOT EXISTS app_settings (
  key text PRIMARY KEY,
  value text NOT NULL,
  updated_at timestamptz NOT NULL DEFAULT now()
);
ALTER TABLE app_settings ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Admin only" ON app_settings USING (true) WITH CHECK (true);
```
